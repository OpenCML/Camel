# Security Policy

[中文简体](SECURITY.cn.md) | English

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Development (main) | Yes |
| Older releases | No longer actively supported |

## Reporting a Vulnerability

If you discover a security vulnerability in Camel, please **do not** discuss it in a public issue.

Please report it privately via:

1. **GitHub Private Security Advisory**  
   Submit via [Camel Security Advisories](https://github.com/OpenCML/Camel/security/advisories/new).

2. **Or contact maintainers**  
   Reach out through the [OpenCML organization](https://github.com/OpenCML).

We will respond within a reasonable time after confirming the report, and will:

- Acknowledge receipt
- Assess severity and impact
- Develop a fix and release a patch when appropriate
- Credit reporters where applicable after the fix is released

## Security Updates

Security fixes are delivered through:

- New releases (tags)
- [GitHub Security Advisories](https://github.com/OpenCML/Camel/security/advisories)

We recommend watching this repository for Release and Security notifications.
