# 安全政策 (Security Policy)

[English](../SECURITY.md) | 中文简体

## 支持的版本

| 版本 | 支持状态     |
|------|--------------|
| 最新发布版 | 支持         |
| 开发版 (main) | 支持       |
| 更早版本 | 不再积极支持 |

## 报告安全漏洞

如果你发现 Camel 项目中存在安全漏洞，请**不要**在公开的 Issue 中讨论。

请通过以下方式私下报告：

1. **GitHub 私有安全通告**
   访问 [Camel Security Advisories](https://github.com/OpenCML/Camel/security/advisories/new) 提交私有安全通告。

2. **或联系维护者**
   通过 [OpenCML 组织](https://github.com/OpenCML) 联系项目维护者。

我们会在确认漏洞后的合理时间内做出响应，并会：
- 确认收到报告
- 评估漏洞的严重程度和影响范围
- 制定修复方案并在合适时机发布补丁
- 在修复发布后（如适用）公开致谢

## 安全更新

安全修复会通过：
- 新的发布版本（tag）
- [GitHub Security Advisories](https://github.com/OpenCML/Camel/security/advisories)

建议关注本仓库的 Release 和 Security 相关通知。
